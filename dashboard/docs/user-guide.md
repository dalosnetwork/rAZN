# User Guide

## Scope
This guide covers the current user experience only.
Legacy pages are intentionally excluded.

## What This Application Is For
This application helps you:
- complete business verification (KYB),
- request mint operations,
- request redemption operations,
- monitor reserve transparency,
- manage wallet connections and addresses,
- manage your account preferences.

## Quick Start
1. Sign in or create an account.
2. Open **KYB** and make sure required documents are complete.
3. Use **Mint** to submit mint requests.
4. Use **Redeem** to submit redemption requests.
5. Track progress from **Overview**, **Mint**, **Redeem**, and **Wallet**.
6. Use **Settings** to maintain profile, notifications, and security preferences.

## Main Navigation (User)
- **Overview**: Your high-level account health and latest activity.
- **Mint**: Create and monitor mint requests.
- **Redeem**: Create and monitor redemption requests.
- **KYB**: Upload/update verification documents and respond to review feedback.
- **Reserve Transparency**: Review reserve/supply snapshots and coverage updates.
- **Wallet**: Connect wallet, manage linked addresses, and track wallet activity.
- **Settings**: Profile, notifications, security sessions, and dashboard preferences.

## End-to-End User Flow
1. **Access your account**
   - Sign in with your credentials or social sign-in.
   - If needed, register first and then continue.
2. **Complete verification**
   - Open **KYB** and upload all required items.
   - Submit updates for review when documents are ready.
3. **Submit a mint request**
   - Enter amount, payment reference, and destination wallet.
   - Submit and monitor request status/timeline.
4. **Monitor holdings and transparency**
   - Use **Overview** for account summary.
   - Use **Wallet** for connected addresses and balances.
   - Use **Reserve Transparency** for reserve coverage visibility.
5. **Submit a redemption request (when needed)**
   - Enter amount, payout details, and optional notes.
   - Follow queue and processing progress.
6. **Keep account settings current**
   - Update profile/contact data.
   - Configure notifications and security controls.

## How To Use Each User Page

### Overview
Use this page as your daily starting point:
- Check current holdings and pending requests.
- Review KYB status and blockers.
- Open quick actions to mint, redeem, KYB, or reserve transparency.

### Mint
Use this page to create mint requests:
1. Fill in amount, source account, payment reference, and destination wallet.
2. Add optional notes.
3. Submit the request.
4. Use search/filter to find previous requests.
5. Open request details to review timeline updates.

### Redeem
Use this page to request payouts:
1. Enter amount and destination account.
2. Select payout route.
3. Add optional notes and submit.
4. Track queue position and status.
5. Review details and timeline for each request.

### KYB
Use this page to maintain verification readiness:
1. Review required checklist items.
2. Upload missing or updated documents.
3. Submit updates for review.
4. Check submission history and reviewer feedback.
5. Update responses when additional documents are requested.

### Reserve Transparency
Use this page to monitor reserve confidence:
- Review latest reserve value, supply, and coverage ratio.
- Check freshness and alert notes.
- Inspect snapshot history for trend and consistency.

### Wallet
Use this page to manage wallet connectivity:
- Connect or disconnect wallet session.
- Add linked addresses.
- Filter/search addresses by network and status.
- Open an address to view details and timeline.
- Track recent wallet activity.

### Settings
Use this page to control your account behavior:
- **Profile**: personal and organization details.
- **Notifications**: alert channels and critical-only behavior.
- **Security**: MFA preference, session timeout, session revocation.
- **Preferences**: landing page, display density, digest preferences.

## Request & Review Status Meanings (User-Friendly)
- **Draft**: started but not submitted.
- **Submitted**: sent for processing.
- **Pending / Under Review**: waiting for operations/compliance action.
- **Approved**: accepted and moving forward.
- **Queued / Processing**: in payout or ops execution flow.
- **Completed**: finished successfully.
- **Needs Update**: more information/documents required.
- **Rejected**: request/case not accepted.
- **Blocked**: action cannot continue until a blocker is resolved.

## Common Situations
- **You cannot mint/redeem**: check **KYB** for missing required documents.
- **A request is delayed**: open details and review latest timeline update.
- **You need audit context**: use page-level history tables and timelines.
- **You signed in on the wrong device**: use **Settings > Security** to revoke sessions.
